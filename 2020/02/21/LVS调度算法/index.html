一、调度算法

主要分为两种

  • 固定算法(静态调度算法): 只考虑算法本身去调度，
    • 消耗资源比较小
  • 动态算法(动态调度算法): 除了考虑算法本身,还要考虑服务器状态
    • 消耗资源比较大

1、静态调度算法

RR(轮询)

将每次用户的请求分配给后端服务器,从第一台服务器开始, 到第 N 台结束,然后再循环

WRR(加权轮询)

按照权重的比例实现多台主机之间进行调度

SH(source hash/源地址散列)

将同一个 IP 的用户请求发到同一个服务器,解决 session 共享等问题

解释: 假设 session 存在 web 服务器上(真实环境 session 最好存在分布式存储上), 而 lvs 后面对应 3 台服务器 s1,s2,s3,此时客户端 c 请求过来访问 s1 的 1.png,s1 的 session 存储了用户 c 信息,等 c 再请求落到 s2 上的时候,s2 上没有该用户信息,而此时,其实 s1 上的 session 有了用户 c 信息,而 SH 算法就可以解决,让该用户 c 一直落到 s1.

DH(target hash/目标地址散列)

将同一个目标地址的请求发送到通一个服务器, 提高缓存命中率

解释: 假设有三台缓冲服务器 SQUID 分别是 s1, s2, s3,一台后端 apache. 此时客户端 c1 访问http://xxx/a/1.png, 第一次访问 s1,s1 没有查找 apache 并返回同时设置缓冲,一会另一个客户端 c2 也访问同样的链接,访问到了 s2, s2 上没有再去访问 apache, 由此看出这是有问题,因为 s1 上已经有了 1.png,而 DH 就是为了解决这个问题, 让任何访问该链接的都落到 s1 上.

2、动态调度算法

在谈动态调度算法之前需要了解http 连接

  • 活动连接：
    • 传输数据
  • 非活动连接：
    • 刚建立连接还没传输数据
    • 传输数据完毕还没来得及断开

LC(lest-connection/最少连接)

将新的连接请求,分配给连接数最少的服务器, 活动连接*256+非活动连接

WLC(加权最少连接)

特殊的最少连接算法,权重越大承担的请求越多, (活动连接*256+非活动连接)/权重

SED 最短期望延迟

特殊的 WLC 算法, (活动属+1) * 256 / 权重

解释: 其实 WLC 和 LC 都存在一个问题, 假设有两台服务器 s1、s2,它们的连接数都是 0, 权重比是 s1:s2 = 1,2 此时来了一个请求,按照 LC 和 WLC 的算法得出的结果都是 0,那么就会分配给第 1 台(按照顺序),但是我们是想分配 s2 的.于是 SED 巧妙的利用算法解决这个问题

NQ(永不排队)

特殊的 SED 算法,无需等待,如果有真实服务器的连接数为 0 那就直接分配不需要运算

LBLC(特殊的 DH 算法)

在 DH 算法中,如果访问量比较大的时候容易把 s1 撑炸,此时再选一个 s2,然 s1 和 s2 之间轮流

LBLCR(LBLC + 缓存)

尽可能提高负载均衡和缓存命中率的折中方案, 其实就是 DH 算法中的缓存之间的同步. 比如说 s1 有缓存了,此时请求落到 s2 上,s2 不到 apache 去查找数据,而是到 s1 去同步缓存
